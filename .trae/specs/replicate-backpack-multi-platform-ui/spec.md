# Backpack 多端前端 UI 复刻 Spec

## Why
当前仓库已具备 Flutter 钱包基础，但缺少与目标参考项目一致的跨端 UI 体系。需要先完整复刻移动端、扩展端与官网桌面端 UI，为后续细节调整提供统一基线。

## What Changes
- 新增跨端 UI 复刻范围与验收标准（Flutter 移动端、浏览器扩展、官网桌面端）。
- 新增统一设计令牌与交互模式约束，确保三端视觉与操作一致性。
- 新增页面级复刻清单与交互矩阵，覆盖主流程与高频入口。
- 新增“先复刻后微调”的执行边界，第一阶段不改动业务规则与后端契约。
- 新增第二阶段目标：完整实现所有交互设计与动效细节，达到可交付展示标准。

## Impact
- Affected specs: `design-flutter-web3-wallet-ui-matrix`、`implement-production-grade-wallet-architecture`
- Affected code: `wallet-app-flutter/`、`wallet-app/`、新增扩展端 UI 模块目录（待实现）

## ADDED Requirements
### Requirement: 跨端 UI 复刻总目标
系统 SHALL 在第一阶段完成目标参考项目的三端 UI 结构复刻，包括 Flutter 移动端、浏览器扩展端和官网电脑端页面。

#### Scenario: 三端范围完整定义
- **WHEN** 开始执行复刻工作
- **THEN** 必须同时包含移动端、扩展端、官网桌面端三个交付面

### Requirement: Flutter 移动端复刻
系统 SHALL 在 Flutter 侧复刻钱包主导航、资产/收藏/活动主视图、发送接收流程、设置与安全确认页面布局与交互节奏。

#### Scenario: 移动端主流程可走通
- **WHEN** 用户从首页进入资产、发送、设置、安全确认流程
- **THEN** 页面跳转、布局结构、关键按钮与状态反馈符合复刻基线

### Requirement: 浏览器扩展端复刻
系统 SHALL 提供扩展端 popup 形态 UI，复刻主容器、顶部/标签导航、核心流程弹层与搜索入口的交互组织方式。

#### Scenario: 扩展端核心导航可用
- **WHEN** 用户在扩展 popup 中执行标签切换、搜索与发送流程入口操作
- **THEN** 导航层级、弹层行为与可见反馈与复刻定义一致

### Requirement: 官网电脑端复刻
系统 SHALL 提供面向电脑端的官网 UI 页面复刻，至少覆盖首页、下载页、信息页的布局与视觉风格。

#### Scenario: 官网桌面端信息可访问
- **WHEN** 用户在桌面浏览器访问官网页面
- **THEN** 页面布局、主要信息区块与下载入口可正常展示与访问

### Requirement: 统一设计令牌与组件映射
系统 SHALL 建立跨端设计令牌映射（颜色、字号、圆角、间距、层级、动效），并在三端组件实现中统一落地。

#### Scenario: 设计一致性检查通过
- **WHEN** 对比三端相同组件类型（按钮、卡片、输入、标签、弹层）
- **THEN** 样式参数与状态表现符合统一映射规则

### Requirement: 复刻验收与差异归档
系统 SHALL 输出页面与交互对照清单，记录“已复刻项、差异项、后续可调项”，作为第二阶段微调输入。

#### Scenario: 第一阶段可交付
- **WHEN** 复刻阶段完成
- **THEN** 存在可审阅的对照清单与差异归档，且不阻断后续细节调整

### Requirement: 交互与动效完整实现
系统 SHALL 在第二阶段完整实现三端交互设计与动效细节，包括页面转场、元素入场、手势/点击反馈、状态切换过渡、模态开合、列表骨架与空态过渡。

#### Scenario: Flutter 端动效完整
- **WHEN** 用户执行导航切换、发送流程、设置切换、安全确认
- **THEN** 页面与组件动效连续、时长统一、无突兀跳变

#### Scenario: 扩展端动效完整
- **WHEN** 用户在 popup 内切换标签、打开搜索、打开/关闭模态、提交表单
- **THEN** 动效节奏与交互反馈与设计令牌一致

#### Scenario: 官网端动效完整
- **WHEN** 用户在桌面端浏览首页、下载页、信息页并滚动与悬停
- **THEN** 区块入场、按钮悬停、页面过渡达到统一动效标准

### Requirement: 交互一致性回归
系统 SHALL 提供三端交互与动效回归检查项，确保更新后不破坏已有流程与可用性。

#### Scenario: 回归通过
- **WHEN** 执行交互回归清单
- **THEN** 关键流程可用、动效触发稳定、无阻断级交互缺陷

## MODIFIED Requirements
### Requirement: Flutter 钱包 UI 基线
现有 Flutter 钱包页面 SHALL 从“功能演示导向”调整为“参考项目复刻导向”，优先保证布局与交互一致性，再进行文案与细节优化。

### Requirement: 第一阶段复刻范围
第一阶段“结构复刻” SHALL 升级为“结构 + 交互 + 动效完整复刻”，未完成项进入当前增量任务并优先实现。

## REMOVED Requirements
### Requirement: 第一阶段进行业务规则重构
**Reason**: 本次目标是先完整复刻前端 UI，不引入额外业务重构风险。  
**Migration**: 业务规则优化与后端契约调整推迟到第二阶段微调任务。
