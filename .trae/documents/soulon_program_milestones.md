# Soulon 项目统一里程碑与交付验收计划

## 1. 计划总览

本计划对 Soulon Deep Chain 与 Soulon Wallet 采用统一里程碑管理，确保链端与钱包端同步交付、同步验收、同步上线。

## 2. 统一里程碑

| 里程碑 | 链端目标 | 钱包端目标 | 联合验收输出 |
| :--- | :--- | :--- | :--- |
| P0 立项冻结 | 技术路线与模块边界冻结 | 产品范围与技术栈冻结 | 立项评审纪要 |
| P1 架构完成 | 链骨架可运行，基础模块接通 | 钱包工程可运行，链连接打通 | 架构评审结论 |
| P2 MVP 可用 | 资产/质押/治理核心模块可用 | 转账/质押/投票主链路可用 | E2E 回归报告 |
| P3 稳定就绪 | 监控告警、参数治理、灾备完成 | 异常恢复、重试机制、安全确认完成 | 稳定性验收报告 |
| P4 主网上线 | 创世发布、验证人接入、网络稳定 | 正式钱包发布与兼容验证 | 上线检查单与复盘报告 |

## 3. 工作分解结构（WBS）

### 3.1 链端 WBS
- WBS-C1：工程初始化与模块装配。
- WBS-C2：创世流程、验证人流程与测试网部署。
- WBS-C3：业务模块参数定稿与集成测试。
- WBS-C4：可观测性、备份恢复、升级与回滚预案。

### 3.2 钱包端 WBS
- WBS-W1：账户体系与本地密钥安全。
- WBS-W2：资产转账链路与交易状态跟踪。
- WBS-W3：质押与治理链路。
- WBS-W4：稳定性治理、发布管道与运维支持。

### 3.3 联合 WBS
- WBS-J1：链钱包版本映射矩阵维护。
- WBS-J2：联调测试与回归测试自动化。
- WBS-J3：发布窗口管理与变更冻结机制。

## 4. 验收机制

### 4.1 阶段验收标准
- 功能验收：需求清单中必须功能全部可用。
- 质量验收：关键路径自动化通过，P0 缺陷为零。
- 运维验收：监控、告警、备份、回滚可演练通过。
- 安全验收：签名流程、密钥存储、依赖漏洞检查通过。

### 4.2 文档验收标准
- 每个里程碑必须提交：变更说明、测试报告、风险列表、发布记录。
- 所有文档必须具备版本号、责任人、更新时间与适用环境标识。

## 5. 依赖与关键路径

### 5.1 关键依赖
- 链端 gRPC/REST 接口稳定性直接影响钱包联调进度。
- 链端参数冻结窗口直接影响钱包手续费与交易策略配置。
- 测试网稳定度直接影响 MVP 验收效率。

### 5.2 关键路径
- 链端模块稳定 → 钱包交易链路稳定 → 联合回归 → 发布冻结 → 主网上线。

## 6. 风险门槛与触发动作

| 触发条件 | 风险级别 | 处理动作 |
| :--- | :--- | :--- |
| 链端接口连续异常 | 高 | 暂停钱包发布，切换备用节点，开启联调排障 |
| 关键交易失败率异常 | 高 | 启动缺陷战情机制，冻结发布窗口 |
| 主网前参数频繁变动 | 中 | 增加参数变更审批并锁定配置发布时间 |
| 回滚演练失败 | 高 | 禁止上线，优先修复回滚链路 |

## 7. 发布治理规则

- 进入发布窗口后仅允许修复阻断缺陷，不接受新需求。
- 链端与钱包端发布版本必须在兼容矩阵中通过验证。
- 发布前必须完成联合 Go/No-Go 评审并留存记录。

## 8. 上线后 30 天运营计划

- 每日跟踪交易成功率、节点健康、钱包崩溃率。
- 每周输出运营周报，包含缺陷趋势与参数优化建议。
- 每两周评审一次下一版本迭代范围与风险。

## 9. 最终交付物总清单

- Soulon Deep Chain 开发与运维文档包。
- Soulon Wallet 开发、测试与发布文档包。
- 联合测试报告与兼容矩阵。
- 主网上线检查单、值班手册、应急预案与复盘报告。
