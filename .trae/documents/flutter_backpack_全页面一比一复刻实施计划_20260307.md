# Flutter Backpack 全页面一比一复刻实施计划

## 目标与范围
- 目标：将 Flutter 端复刻模块从“仅首页近似”升级为“所有复刻页面与 Chrome 扩展 UI 一比一对齐（结构、文案、状态、交互节奏）”。
- 复刻范围（当前路由）：
  - `/replica/mobile/home`
  - `/replica/mobile/send`
  - `/replica/mobile/receive`
  - `/replica/mobile/settings`
  - `/replica/mobile/security-confirm`
- 基准来源：本仓库 `wallet-extension/src/popup/main.ts` 与 `wallet-extension/src/styles/popup.css`（以扩展 UI 为单一视觉与交互基准）。

## 对齐原则（强约束）
- 只对齐扩展 UI，不再参考官网页样式。
- 每页执行“三层对齐”：
  - 结构层：区块顺序、分栏关系、按钮组关系一致。
  - 视觉层：字号/字重/间距/圆角/边框/色值/阴影一致。
  - 交互层：激活态、禁用态、空态、错误态、成功态、动效节奏一致。
- 文案优先按截图与扩展 UI 使用中文；技术占位文案全部清理。
- 页面差异以“像素收敛清单”逐项归零，不留“后续微调”口头项。

## 页面映射与实施步骤

### 步骤 1：建立全页面映射矩阵（先做清单再改代码）
- 输出“Flutter 页面区块 ↔ 扩展 UI 区块”的一一映射表，覆盖 5 个复刻页面。
- 对每页列出：
  - 必有区块（Header、Tab、Card、Form、BottomNav、Modal 等）
  - 必有状态（loading/empty/error/success/active）
  - 必有交互（点击、切换、路由跳转、输入校验、反馈提示）
- 产出可执行差异项列表，按“高影响优先”排序。

### 步骤 2：首页 `/replica/mobile/home` 深度收敛
- 对齐区块：顶部账户条、一级 tab、余额区、四宫格快捷操作、安全横幅、轮播点、资产卡、管理代币显示、底部导航。
- 对齐细节：
  - 文案与图标语义一致（法币/接收/发送/兑换、资产总览/兑换/探索）。
  - 金额层级（主金额、副金额、涨跌）与字体层级一致。
  - 选中态与未选中态颜色、背景、边框一致。
- 对齐交互：
  - tab 切换内容与动效方向一致。
  - 快捷操作跳转到对应复刻页面。
  - 底部导航切换时保留页面状态与动效。

### 步骤 3：发送页 `/replica/mobile/send` 一比一复刻
- 对齐区块：收款地址输入、金额输入、步骤提示、确认区、提交结果区。
- 对齐状态：
  - 地址为空/非法、金额为空/非法、余额不足、提交成功、提交失败。
- 对齐交互：
  - “下一步/提交”流程与按钮状态切换一致。
  - 错误反馈文案与展示位置一致。
  - 提交成功后结果信息与哈希展示节奏一致。

### 步骤 4：接收页 `/replica/mobile/receive` 一比一复刻
- 对齐区块：收款金额、备注、二维码/请求链接展示区、复制与分享动作。
- 对齐状态：
  - 未生成、已生成、生成失败、参数校验失败。
- 对齐交互：
  - 生成请求后结果区动效与信息层级一致。
  - 复制/分享反馈一致（toast 或行内反馈按扩展风格对齐）。

### 步骤 5：设置页 `/replica/mobile/settings` 一比一复刻
- 对齐区块：设置项分组、开关项、跳转项、危险操作区。
- 对齐状态：
  - 开关 on/off、禁用项、危险操作确认前后。
- 对齐交互：
  - 开关变化即时反馈。
  - 跳转项行为与层级一致。
  - 危险操作二次确认流程与文案一致。

### 步骤 6：安全确认页 `/replica/mobile/security-confirm` 一比一复刻
- 对齐区块：PIN 输入、生物识别状态、风险提示、确认按钮、结果反馈。
- 对齐状态：
  - PIN 错误、生物识别未完成、双因子通过、拒绝状态。
- 对齐交互：
  - 按钮禁用/可用切换条件一致。
  - 错误与成功反馈位置、颜色、文案一致。

### 步骤 7：全局组件与设计令牌统一
- 在不破坏现有主题体系前提下，统一复刻页专用样式令牌：
  - 色板（背景/表面/边框/强调/文本）
  - 字体层级（标题/正文/标签/辅助）
  - 间距与圆角（8/10/12/14/16 等）
  - 动效（fast/normal，入场与切换曲线）
- 清理复刻页中与扩展风格冲突的历史组件样式。

### 步骤 8：路由与导航一致性收尾
- 确认默认入口仍为复刻首页。
- 确认主页快捷操作、底部导航、设置入口、安全入口路由互通。
- 确认回退行为（pageBack）与扩展预期一致，不出现死路由。

### 步骤 9：测试体系更新（先修断言再做回归）
- 更新以下测试以反映“一比一复刻”后的真实结构与文案：
  - `test/task3_replica_mobile_ui_test.dart`
  - `test/task7_replica_motion_regression_test.dart`
  - `test/wallet_app_smoke_test.dart`
- 为 5 个复刻页补齐关键断言：
  - 结构断言（关键区块存在）
  - 路由断言（关键跳转可达）
  - 状态断言（错误/成功/空态）
  - 交互断言（按钮状态、tab 切换、提交流程）

### 步骤 10：验收与像素收敛闭环
- 执行：
  - `flutter analyze --no-fatal-infos`
  - `flutter test`
- 开启预览进行人工比对（以扩展 UI 为准）：
  - 记录每页“未对齐点清单”（尺寸、间距、颜色、图标、文案）
  - 逐项修正直至“清单为 0”
- 最终交付：页面逐项对齐结果 + 已通过验证结果。

## 实施顺序（严格执行）
1. 映射矩阵与差异清单  
2. 首页收敛  
3. 发送页  
4. 接收页  
5. 设置页  
6. 安全确认页  
7. 令牌与组件统一  
8. 路由收尾  
9. 测试更新  
10. analyze/test + 人工像素验收

## 完成标准（DoD）
- 5 个复刻页面全部具备扩展 UI 对齐结构与关键交互。
- 不存在明显占位样式、占位文案或未实现交互入口。
- `flutter analyze --no-fatal-infos` 无 error。
- `flutter test` 全量通过。
- 人工对比清单归零（无剩余“后续优化项”）。
