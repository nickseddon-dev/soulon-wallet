# Soulon Wallet 开发计划（执行版）

## 1. 目标与版本范围

### 1.1 目标
- 交付面向 Soulon Deep Chain 的非托管钱包 MVP。
- 覆盖账户、资产、质押、治理四大业务闭环。
- 保证钱包版本与链版本的兼容演进。

### 1.2 版本范围
- V1（MVP）：Web 钱包 + Cosmos 原生能力。
- V1.5：稳定性与安全加固。
- V2：多终端扩展与硬件钱包增强支持。

## 2. 架构与分层

### 2.1 客户端分层
- UI 层：页面状态、交互流程、表单校验。
- 业务层：账户、转账、质押、治理用例编排。
- 链适配层：CosmJS 调用封装、节点路由、错误映射。
- 安全层：本地加密、会话锁、签名前确认。

### 2.2 关键接口
- 只读接口：余额、区块高度、交易记录、提案详情、验证人列表。
- 写入接口：转账广播、委托/撤销、奖励提取、投票提交。
- 状态接口：交易广播后轮询确认与超时重试。

## 3. 功能拆解与实施顺序

### 3.1 模块 A：账户中心
- 创建钱包：助记词生成、确认、加密保存。
- 导入钱包：助记词与私钥导入校验。
- 账户管理：多账户切换、地址复制、导出保护。
- 验收标准：账户可稳定恢复且地址一致性通过校验。

### 3.2 模块 B：资产管理
- 资产首页：主币余额、可用余额、预估手续费。
- 转账流程：构建交易、签名确认、广播、回执展示。
- 交易历史：按账户分页查询与状态筛选。
- 验收标准：转账成功率与状态一致性达标。

### 3.3 模块 C：质押收益
- 验证人列表：佣金率、在线状态、累计投票权展示。
- 委托操作：输入金额、风险提示、签名提交。
- 奖励领取：单验证人与全部奖励领取。
- 验收标准：委托、赎回、领奖励全链路可复现。

### 3.4 模块 D：治理投票
- 提案广场：状态分类、提案详情、关键字段展示。
- 投票流程：赞成/反对/弃权/否决，签名与上链确认。
- 记录追踪：个人投票记录与提案最终结果。
- 验收标准：投票结果与链上状态一致。

## 4. 工程计划与迭代节奏

| 迭代 | 目标 | 主要交付 |
| :--- | :--- | :--- |
| Sprint 1 | 工程底座 | 项目初始化、网络配置、账户模块雏形 |
| Sprint 2 | 资产闭环 | 余额查询、转账构建签名广播、交易回执 |
| Sprint 3 | 质押闭环 | 验证人列表、委托与奖励领取 |
| Sprint 4 | 治理闭环 | 提案浏览、投票与记录追踪 |
| Sprint 5 | 稳定加固 | 错误处理、重试、监控埋点、性能优化 |
| Sprint 6 | 发布准备 | 回归测试、安全检查、上线发布包 |

## 5. 数据与配置规范

### 5.1 链配置
- chainId、bech32Prefix、gasPrice、rpc/grpc/rest 端点统一配置管理。
- 支持环境隔离：dev / testnet / mainnet。

### 5.2 交易配置
- 默认 gas 策略 + 手续费级别档位。
- 广播模式、确认超时、重试间隔可配置。

### 5.3 安全配置
- 本地存储加密算法与参数固定版本化。
- 连续失败解锁阈值与会话自动锁定策略。

## 6. 质量保障计划

### 6.1 测试矩阵
- 单元测试：业务函数、签名流程、地址处理。
- 集成测试：链适配层与节点交互。
- E2E 测试：账户创建到交易上链全流程。

### 6.2 门禁规则
- 关键业务路径自动化通过率必须达标。
- 高优先级缺陷关闭后方可进入发布阶段。
- 每个版本必须完成链兼容性回归。

## 7. 安全与合规计划

- 助记词页面禁止截屏提示与风险声明。
- 交易签名前展示完整交易摘要并二次确认。
- 对高风险地址和异常金额触发额外确认流程。
- 发布前完成依赖安全扫描与已知漏洞核查。

## 8. 上线与运营支持

### 8.1 上线准备
- 生成发布构建产物与版本清单。
- 完成回滚包与应急说明文档。

### 8.2 上线后保障
- 接入关键行为埋点与错误监控。
- 建立用户问题分类响应机制。
- 发布后一周进行稳定性复盘与修复迭代。

## 9. 对链端协同依赖

- 依赖链端提供稳定的 gRPC 与 REST 接口。
- 依赖链端确定主网参数与手续费策略。
- 依赖链端治理、质押模块参数冻结窗口。

## 10. 交付验收清单

- 钱包可完成账户创建、导入、备份恢复。
- 钱包可完成转账并可追踪上链结果。
- 钱包可完成质押委托与奖励领取。
- 钱包可完成治理投票并查询投票结果。
- 测试报告、发布文档、回滚文档齐备。
