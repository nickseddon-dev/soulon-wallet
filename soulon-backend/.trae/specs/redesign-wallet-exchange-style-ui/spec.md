# 钱包简洁交易所风格 UI 改造 Spec

## Why
当前钱包页面以工程可用为主，信息密度与视觉层次已具备基础能力，但缺少“简洁交易所风格”的统一视觉语言与交互节奏。需要在不改变核心业务流程的前提下，提升可读性、专业感与操作效率。

## What Changes
- 建立统一视觉基线：深色主界面、信息卡片层级、按钮/输入框一致化
- 重构全局布局：顶部信息区、导航区、内容区分层清晰
- 改造登录、状态、事件、详情页面为简洁交易所风格
- 强化列表可读性与交互反馈：筛选区、表格、分页、状态文案
- 补充响应式规则，保障常见桌面与窄屏窗口下可用性

## Impact
- Affected specs: 钱包前端展示体验、交互一致性、可读性
- Affected code: `wallet-app/src/index.css`、`wallet-app/src/App.tsx`、`wallet-app/src/pages/*.tsx`

## ADDED Requirements
### Requirement: 统一交易所风格视觉系统
系统 SHALL 提供统一的简洁交易所风格视觉规则，包括色板、间距、圆角、字体层级与控件样式。

#### Scenario: 访问任意页面
- **WHEN** 用户进入钱包任意页面
- **THEN** 页面在配色、控件与排版上呈现一致的交易所风格

### Requirement: 页面结构与信息层级优化
系统 SHALL 重构核心页面的信息层级，保证主要信息与主要操作在首屏可快速识别。

#### Scenario: 查看状态与事件
- **WHEN** 用户进入链状态或事件页面
- **THEN** 用户可快速分辨筛选区、数据区、操作区与反馈区

### Requirement: 可用性与响应式保障
系统 SHALL 在常见窄屏场景保持可操作性，包括换行、滚动与按钮布局可用。

#### Scenario: 窄窗口访问
- **WHEN** 用户缩小浏览器窗口
- **THEN** 页面布局自适应且不出现关键操作不可达

## MODIFIED Requirements
### Requirement: 钱包页面展示风格
钱包页面展示从“工程默认样式”修改为“简洁交易所风格统一样式”，业务逻辑与接口契约保持不变。

## REMOVED Requirements
### Requirement: 仅满足可运行的基础视觉样式
**Reason**: 无法满足正式对外使用时的品牌一致性与操作效率要求。  
**Migration**: 迁移到统一视觉基线与页面级样式规范，并通过前端门禁验证。
